📚 PKCE란?
Public Client는 코드가 브라우저에 노출되어 client_secret을 안전하게 보관할 수 없습니다. PKCE는 이를 해결하기 위해 동적으로 생성된 Code Verifier와 Code Challenge를 사용합니다.

🔍 PKCE 생성 과정

1️⃣ Code Verifier (랜덤 문자열 43~128자)
2️⃣ Code Challenge (SHA256 해싱)
3️⃣ Authorization 요청 시 Challenge 전송 →
4️⃣ Token 요청 시 Verifier 전송 →
5️⃣ 서버가 SHA256(Verifier) == Challenge 검증 ✅
